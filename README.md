# Finapp

Finapp — модульное Android-приложение для учёта личных финансов. Приложение помогает вести расходы, доходы, счета и категории в одном месте, а кодовая база разделена на независимые feature- и core-модули.

Проект сделан как портфолио-приложение с акцентом на архитектуру, работу с локальными данными, синхронизацию с backend API и чистое разделение ответственности между UI, data layer, локальным хранилищем, сетью и фоновыми задачами.

## Возможности

- Учёт расходов и доходов
- Просмотр истории транзакций
- Редактирование существующих доходов и расходов
- Экраны анализа для доходов и расходов
- Управление информацией о счёте
- Работа с тегами/категориями транзакций
- Настройка темы приложения и основного цвета интерфейса
- Локальное хранение данных
- Синхронизация транзакций с backend API
- Фоновая синхронизация с учётом состояния сети и повторными попытками при ошибках

## Технологии

- **Kotlin**
- **Jetpack Compose** и **Material 3** для интерфейса
- **Navigation Compose** для навигации между экранами
- **Dagger 2** и KSP для dependency injection
- **Coroutines** для асинхронной работы
- **Retrofit** и **OkHttp** для работы с API
- **Room** для локальной базы данных
- **DataStore** для хранения настроек и служебных данных синхронизации
- **WorkManager** для фоновой синхронизации
- **Gradle Kotlin DSL**, version catalog и custom convention plugins

## Архитектура

Finapp построен как multi-module Android-проект. Такой подход помогает держать кодовую базу более понятной, изолировать фичи друг от друга и упростить развитие приложения.

```text
Finapp
├── app                 # Точка входа, DI-граф, тема приложения, настройка WorkManager
├── core
│   ├── common          # Общие утилиты и базовые абстракции
│   ├── data            # Репозитории и маппинг моделей
│   ├── database        # Room database и локальные источники данных
│   ├── remote          # Retrofit API и сетевые источники данных
│   ├── settings        # Хранение настроек приложения
│   └── work            # Фоновые задачи синхронизации
└── feature
    ├── account         # Экран счёта и редактирование данных
    ├── charts          # UI для графиков
    ├── common          # Общие UI-компоненты и тема
    ├── expenses        # Расходы, история, редактирование и анализ
    ├── home            # Главный контейнер и нижняя навигация
    ├── income          # Доходы, история, редактирование и анализ
    ├── settings        # Настройки, выбор цвета и экран About
    └── tags            # Экран тегов/категорий
```

### App layer

Модуль `app` объединяет приложение: создаёт корневой Dagger-компонент, применяет тему, инициализирует навигацию и запускает фоновую синхронизацию.

### Feature layer

Каждая feature отвечает за свой пользовательский сценарий, навигацию и ViewModel. Главный экран содержит bottom navigation между расходами, доходами, счётом, тегами и настройками.

### Data layer

Data layer предоставляет репозитории для feature-модулей и скрывает детали реализации за интерфейсами. Репозитории объединяют работу с backend API и локальной базой данных, а также отвечают за преобразование DTO/database-моделей в модели приложения.

### Локальное хранение и синхронизация

Транзакции могут сохраняться локально и синхронизироваться с backend API. Несинхронизированные записи обрабатываются через WorkManager: задача запускается при наличии сети, выполняет отправку локальных изменений и повторяется при ошибке.

## Запуск проекта

### Требования

- Android Studio
- JDK 17+
- Установленный Android SDK

### Клонирование репозитория

```bash
git clone https://github.com/kostoev06/Finapp.git
cd Finapp
```

### Настройка local.properties

Проект читает API-данные из `local.properties`. Перед сборкой нужно добавить:

```properties
API_KEY=your_api_token
ACCOUNT_ID=your_account_id
```

Реальные ключи и идентификаторы не должны попадать в репозиторий.

### Сборка

```bash
./gradlew assembleDebug
```

Также проект можно открыть в Android Studio и запустить конфигурацию `app`.

## Что стоит посмотреть в коде

- Multi-module структура с разделением на `core` и `feature`
- Dagger root component и отдельные component builders для feature-модулей
- Repository layer, который координирует remote и local data sources
- Room database для кэширования счетов, категорий и транзакций
- WorkManager worker для синхронизации транзакций и retry-логики
- Compose-навигация с отдельными flow для expenses, income, account, tags и settings

## Статус проекта

Основной сценарий учёта финансов реализован: расходы, доходы, история, редактирование, настройки, локальное хранение и синхронизация с backend API. Авторизация, onboarding и production release pipeline не входят в текущий scope проекта.
